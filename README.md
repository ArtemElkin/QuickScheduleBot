# Quick Schedule Bot

**Telegram-бот для преподавателей и репетиторов**, которые ведут занятия онлайн. Бот берёт на себя рутину: напоминания ученикам, подтверждение и перенос уроков, учёт часовых поясов, приглашение по ссылке. Учитель управляет расписанием и учениками из Telegram; ученики получают уведомления и могут подтвердить или перенести урок без лишней переписки.

## Основные возможности

**Для преподавателей:**
- Регистрация и онбординг
- Ученики: добавление, приглашение в бота по ссылке, карточка с настройками
- Расписание: уроки по дням недели и времени, свободные слоты для переносов
- Напоминания: настраиваемое время (за 15 / 30 / 60 минут и за день до урока в 20:00)
- Напоминания об оплате ученикам (отложенная отправка с произвольным текстом)
- Подписки: несколько тарифов, оплата, автопродление, смена тарифа с перерасчётом
- Реферальная программа: персональный код, скидки за приглашённых (до 50%)
- Группы учеников для групповых занятий и рассылок
- Профиль, часовой пояс, помощь и документы в боте

**Для учеников:**
- Напоминания о предстоящих уроках
- Подтверждение урока или перенос на свободный слот одной кнопкой

**Инфраструктура:**
- Отдельный админ-бот: статистика, логи, мониторинг, алерты, выдача подписок

## Стек и технологии

| Область | Технологии |
|--------|------------|
| Backend | Python 3.10+, aiogram 3.x (async Telegram Bot API), SQLAlchemy 2.x |
| БД | PostgreSQL (production), опционально SQLite для разработки |
| Платежи | Интеграция с платёжной системой; отдельный webhook-сервис на FastAPI |
| Фоновые задачи | APScheduler — напоминания, проверка подписок, отложенные уведомления |
| Часовые пояса | pytz для корректной работы расписания и напоминаний в разных регионах |
| Окружение | Конфигурация через переменные окружения, systemd для деплоя |

## Архитектура (высокоуровнево)

Приложение построено по слоёной архитектуре: запрос из Telegram не попадает в базу напрямую, а проходит через чёткие слои с одной зоной ответственности.

<!-- **Изображение 5 (опционально):** диаграмма тех же слоёв, нарисованная в draw.io / Excalidraw / Mermaid и экспортированная в PNG. Можно заменить блок с ASCII-схемой ниже на одну картинку. -->

```
                    Telegram Bot API
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  Точка входа (Dispatcher)                                │
│  Регистрация роутеров, глобальная обработка ошибок       │
└───────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│  Middleware                                              │
│  Сессия БД · Определение роли (учитель/ученик) ·         │
│  Внедрение сервисов · Проверка подписки                  │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│  Handlers (обработчики событий)                          │
│  Обработка команд, кнопок, сообщений; только вызов       │
│  сервисов и формирование ответа пользователю              │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│  Services (бизнес-логика)                                │
│  Правила подписок, платежей, рефералов, расписания,      │
│  напоминаний; координация между сущностями                │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│  Repositories (доступ к данным)                          │
│  CRUD-операции с БД, без бизнес-правил                   │
└────────────────────────────┬─────────────────────────────┘
                             │
                             ▼
                    PostgreSQL
```

**Принципы:**
- **Handlers → Services → Repositories → Database** — строгий порядок вызовов; хендлеры не обращаются к репозиториям напрямую.
- **Dependency injection** — сервисы и сессия БД передаются через middleware, что упрощает тестирование и подмену зависимостей.
- **Платежи** — вынесены в отдельный webhook-сервис; бот не хранит платёжные секреты и получает только факт успешной оплаты.
- **Планировщик** — живёт рядом с ботом, ставит и снимает задачи напоминаний при изменении расписания и учеников.

Тесты разделены на unit (сервисы, репозитории, хендлеры с моками) и интеграционные (БД, планировщик); миграции схемы — в виде SQL-скриптов.
